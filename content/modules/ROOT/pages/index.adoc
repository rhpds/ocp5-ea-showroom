== Introduction

Welcome to the *OpenShift 5 Early Access Program*. You have been given access
to a fully provisioned OpenShift 5 cluster with several new and updated
features pre-installed and configured.

As part of this EAP, you are being asked to evaluate these features, exercise
the scenarios described in the program materials, and provide structured
feedback to the product teams. Your hands-on testing and observations are
critical to shaping the GA releases of these technologies.

This environment covers five main feature areas:

* *Agentic Troubleshooting* — Automated root-cause analysis and remediation of application issues triggered by alerts
* *OpenShift Lightspeed Agentic Updates* — AI-assisted cluster operations using the new agentic operator, agents, and approval policies
* *MCP Server* — Model Context Protocol servers for OpenShift and ACS, enabling AI agents to interact with cluster and security APIs
* *MCP Gateway* — Part of Red Hat Connectivity Link, providing federated MCP access, team isolation, authentication, and policy enforcement
* *Advanced Cluster Security (ACS) 5* — VM scanning via roxagent, file activity monitoring, MCP server integration, and initContainer scanning

== Using This Environment

Your environment is split into two panels:

* *Left panel* — these instructions. Use the navigation on the left side to move between exercises and appendices.
* *Right panel* — a terminal that is already logged in to your OpenShift cluster as `cluster-admin`. You can run `oc` commands directly here without any additional authentication.

=== Running Commands

Throughout these instructions, you will see command blocks like this:

[source,bash,role="copypaste"]
----
oc get pods -n openshift-lightspeed
----

Click the *copy icon* in the top-right corner of any command block to copy it to your clipboard, then paste it into the terminal on the right.

TIP: You can also type commands directly into the terminal. The terminal supports standard shell features like command history (kbd:[Up] / kbd:[Down] arrows), tab completion (kbd:[Tab]), and kbd:[Ctrl+C] to cancel a running command.

=== Accessing the OpenShift Console

Open the OpenShift web console in a new browser tab using the link in the Credentials table above. You will use the console for UI-based exercises (viewing alerts, managing virtualization, reviewing ArgoCD applications, etc.).

=== Using Your Own Tools

If you prefer to use your own local tools (e.g., `oc` CLI from your laptop, IDE extensions, or AI clients like Claude Desktop or Cursor), you can obtain login credentials from the OpenShift console:

1. Click your username in the top-right corner of the OpenShift console
2. Select *Copy login command*
3. Use the displayed `oc login` command on your local machine

You have full `cluster-admin` access, so you are free to explore beyond the guided exercises.

CAUTION: This environment is managed by OpenShift GitOps (Argo CD). If you change configurations that ArgoCD manages, they will be automatically reverted within a few minutes. If you need to make persistent changes for testing, you can pause auto-sync on the relevant ArgoCD application first (see xref:byom.adoc[Appendix A] for an example).
